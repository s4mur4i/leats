Network

Internal network

This network is considered as the internal enterprise network, which data communication should be trusted. All middlewares used during the problem solving should be accesible on this network.

Network: 1.1.1.0
Netmask: 255.255.255.0
DNS: 1.1.1.1
DHCP: 1.1.1.1
Netmask: 255.255.255.0
Desktop IP: 1.1.1.1
Server IP: 1.1.1.2

Cracker network

This network is considered as unsecure network where potential dangers may come from. It is not a solution to only allow middlewares to listen on the required interface, you have to disallow connections from this network with firewall rules, or with middleware security options.

Network: 2.2.2.0

Firewall

Iptables

Current testing methods use iptables solutions, later on we might implement different solutions aswell.

Selinux

Selinux may never be disabled. it must be always set to enforcing.

Yum repositary

Yum repositary is provisioned with all required packages. Do not use any external or third party repositarys. They might break the system.

Repo: 1.1.1.1

Reinstall
There might be some requirments, or unforseen problems that might require you to reinstall the server. you can reinstall the server with the alts-guest.pl script.
